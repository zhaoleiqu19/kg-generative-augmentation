# 阶段 2 — Spec:现状(stage2-state-of-the-art 中文版)

> 脊柱 stage 2:把一个诊断翻译成一份*生成规格*——为填上诊断出的缺口,要合成什么(哪些样本、多少张、在什么条件下)。

## 是什么
诊断(stage 1)与合成(stage 3)之间的桥。它有两半:
- **决策** —— *哪些*样本要增强、增*多少*(针对/选择问题)。
- **编码** —— 把目标表达成生成器能消费的条件(prompt、layout、几何、region)。

## 为何重要
一份好规格让生成变得*针对*,而针对是唯一可靠有用的区间(参 合成数据有用吗);盲生成浪费且可能有害。规格还设定了诊断所要求的合成与真实之比。

## 现状

### 决策:生成什么 / 生成多少
- ActGen(Huang 2024)—— 用模型的**错分类样本**作为信号;生成更难的相似样本;用远少的图像得到更好结果。最清晰的诊断→spec 链(分类)。
- TADA(Nguyen 2025)—— 只增强**难学子集**(~30–40%)→ **最高 +2.8%**,胜过全量增强;理论上,针对会在不放大噪声的前提下令特征学习速度均质化。"针对 > 盲"的定量论据。

### 编码:生成器能消费的规格格式
- GeoDiffusion(Chen 2023)—— 规格 = **几何/layout** 条件(边界框、相机视角)转成 prompt;首个证明 L2I 扩散数据有助检测器。定义了检测的可执行规格语言。
- ReCon(Zhu 2025)—— **region 可控**生成,通过感知反馈纠偏 + region 对齐交叉注意力做到*忠实*。在遵守 region 规格上是现状最佳(NeurIPS'25 spotlight)。

## 已解决 vs 开放
- **已解决:** 对*分类*,决策一半已被演示——针对困难/错分类子集胜过盲,有数字(ActGen、TADA)。对*检测*,编码一半已存在——layout/region 条件是一个可执行且有益的规格格式(GeoDiffusion、ReCon)。
- **开放(楔子):** **没人把两者为检测连起来。** 把一个诊断出的*检测*失败切片(stage 1;GH-ESD/HiBug2)→ 一个自动的 *layout/region* 规格(GeoDiffusion/ReCon 格式),基本无人涉及。选择准则全是分类图像级(错分类 / 没早学会),而非目标/切片级。(→ GAPS G4)

## 我们的看法
零件从两侧夹住了缺口:检测诊断正在出现(stage 1),可控检测生成已成熟(这里的编码)。未造的桥——*把一个有定位的检测切片自动翻译成 region/layout 生成规格,还带一个有原则的数量*——是整个闭环里最"缺口形状"的开口,且正落在锚点上。
