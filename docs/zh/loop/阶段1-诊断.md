# 阶段 1 — 诊断:现状(stage1-state-of-the-art 中文版)

> 脊柱 stage 1:检测一个视觉模型*在哪里、为什么*失败,粒度要能驱动针对性生成。对我们的区间,暂定是**最薄弱的环节**(见 MAP)。

## 是什么
**错误切片发现 / 失败归因**:把模型的错误变成连贯、命名、以人类术语描述的子群("切片")——这就是告诉 stage 2/3 *生成什么*的信号。该领域分两种范式:
- **先切后标** —— 在嵌入空间里聚类失败,再描述(Domino、Spotlight、FACTS)。
- **先标后切** —— 先生成任务属性/标签(由人或 VLM/LLM),再把数据分组成切片(HiBug2、GH-ESD、LADDER)。

## 为何重要
没有正确粒度的诊断,生成就是盲的——而盲/迭代生成正是*有害*的那个区间(参 合成数据有用吗、Zhang 2024 偏见放大)。诊断是让整个闭环变得针对、安全、高效的杠杆。

## 现状

### 基础(分类)
- Domino(Eyuboglu 2022)—— 定义切片发现;嵌入→切片→以自然语言命名描述。在 1,235 个切片中找到 36%(+12 个百分点);仅逐图分类。

### 诊断 → 生成 的桥
- Chegini(2023)—— 自动描述失败(LLM/VLM)→ 扩散生成针对性修复;跨 40 个模型,**困难子群准确率 ~21% 增益**。分类 / 背景伪相关。
- SafeFix(Ouyang 2025)—— 完整闭环:归因失败 → 条件 T2I 生成 → VLM 过滤 → 重训。类别级。

### 走向检测粒度(缺口)
- GH-ESD(Zhang 2026)—— **实例级**切片发现,用于检测/分割;有定位/关系的切片;Precision@10 0.73 vs 0.63。最接近 G2。
- HiBug2(Chen 2025)—— 先标后切,跨分类/姿态/**检测**;能预测验证集*之外*的错误切片。

## 已解决 vs 开放
- **已解决:** 对*分类*,自动切片发现 + 自然语言描述已成熟,诊断→生成桥已被演示(Chegini ~21%)。
- **开放(我们的楔子):**
  1. 检测粒度诊断(逐目标、有空间定位)才刚出现(GH-ESD、HiBug2),且未在**小/遮挡**目标上证明。(→ G2)
  2. 把一个被发现的检测切片变成生成*规格*(stage 2)基本无人涉及——多数工作走"切片 → 盲重训",而非"切片 → 可控合成"。
  3. 在没有带标注留出集的*部署*画面上诊断(电梯 CCTV)——HiBug2 的"超出验证集"思路是唯一线索。

## 我们的看法
诊断机器已经存在,但是分类形状的。把有定位的、实例级的切片发现推到小目标电梯检测上——并把它的输出接进一个可控生成规格——是 GAPS G1/G2 瞄准的、具体而服务不足的开口。
