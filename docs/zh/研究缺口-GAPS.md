# 候选研究缺口(GAPS 中文版)

> **这是项目的核心交付物。** 每个条目一个候选:缺口 · 证据 · 为何尚未解决 · 能否在锚点上演示(y/n/未知)。论文引用见 [论文索引.md](论文索引.md)。

## G1 — 面向*检测*(而非仅分类)的诊断驱动针对性增强

- **缺口:** "诊断失败 → 生成针对性数据 → 过滤 → 重训"这个闭环主要在图像**分类**上被演示;它向**目标检测**(逐目标、逐属性的失败)的迁移是开放的。
- **证据:** Chegini(诊断→扩散生成,困难子群 ~21%,**分类**)、SafeFix(以分类为中心的修复闭环)、stage4 现状(accumulate/verify/active-loop 等护栏全非检测)、相关系统节点(已梳理最近邻)。
- **最近邻(及为何不达标):** LTDA-Drive(generate→insert→LLM 过滤,用于**检测**,KITTI 稀有类 +34.75%——但由*盲长尾*驱动,不是被实测诊断牵引;且是 3D 自动驾驶);GALOT(真正的 **AL 准则→生成**闭环——但是分类、纯合成、文本伪标签、无框)。
- **为何尚未解决:** 最强的端到端诊断→生成→修复结果(Chegini ~21%;SafeFix)都是分类 + 伪相关;闭环护栏(Gerstgrasser、Yi、Kessler)是 LM/VLM;最近的*检测*闭环(LTDA-Drive)不是诊断*驱动*的,最近的*闭环*(GALOT)不是检测。**没有已发表的闭环把"有据可循的逐目标诊断→spec→合成→过滤后重训"接起来用于目标检测。** 检测失败是局部的(框级),所以归因、验证器、指标(尺寸相关 AP)都和分类修复不同。
- **能在锚点上演示?** 很可能 y —— 电梯电动车检测是一个有清晰失败模式(门口遮挡、小尺度)的检测任务。

## G2 — 面向小/遮挡目标的检测粒度、空间定位失败诊断

- **缺口:** 切片发现对分类已成熟,但对实例级检测才*刚刚出现*,且未在小/遮挡区间验证。两个具体空洞:(a) 有定位的实例级切片对*小*目标管用吗?(b) 一个被发现的检测切片能否被转成一个生成 **spec**(stage 2),而不是触发全量重训?
- **证据:** Domino(成熟但仅分类)、GH-ESD(实例级,P@10 0.73 vs 0.63——最新,未在小目标上测试)、HiBug2(多任务含检测;能预测验证集之外的切片)、小目标检测综述、stage1 综述。
- **为何尚未解决:** 诊断(stage 1)是检测里最不成熟的一环;逐图属性切片对框级失败太粗,而切片→spec 交接(stage 2)基本没人碰。
- **能在锚点上演示?** 很可能 y —— 在电梯检测器上跑 GH-ESD/HiBug2 式实例切片,看切片是否映射到可解释的困难案例(门口遮挡、低光、小尺度)。

## G4 — 切片→spec 桥:把诊断出的*检测*切片自动翻译成 layout/region 生成规格

- **缺口:** 针对(targeting)对分类已解决(选错分类的 / 没早学会的图像),可控检测生成也成熟(layout/region 条件),但**没人把两者连起来**:把一个有定位的检测失败切片变成一个自动的 layout/region 规格——还带一个有原则的数量——无人涉及。
- **证据:** ActGen(错分类→生成,图像级)、TADA(针对性子集 +2.8%,图像级)、GeoDiffusion + ReCon(检测可执行的 layout/region 规格格式)、Scene-Aware(学习式**放置规格**——预测物体现实地该放*哪*,+1.4 mAP——但来自场景先验,非诊断)、DIAG(**人**标区域 = 手工版切片→spec 标记,+18–28% AP)、stage2 综述。
- **为何尚未解决:** 选择准则全是分类图像级;规格编码工作假设 layout 是*给定的*(来自真实标注)或放置先验,而非*从诊断推导*;有推导出 spec 的地方(DIAG)是人标的,不是检测器失败。两套文献不会自动相遇。
- **能在锚点上演示?** y —— 流水线:在电梯检测器上 GH-ESD/HiBug2 切片 → 为失败条件发出 GeoDiffusion/ReCon 式 layout 规格 → 用现有 Flux 流水线生成 → 在切片上量尺寸相关 AP。

## G3 — 生成能否填上*小/遮挡*域差,且在什么比例/过滤下?

- **缺口:** 不清楚哪种扩散增强类型 + 合成与真实之比 + 过滤(基于 VLM?)真能为小/遮挡目标提升 mAP,而非因伪影反伤。
- **证据:** 合成数据有用吗(有用:Azizi 64.96%→69.24% 靠保真度,Ge 检测 ≈ 真实;有害:Zhang 偏见放大)、stage3 现状(ODGEN 遮挡下 +25.3% mAP;AeroGen 稀有类 +12.6–17.8%;X-Paste 长尾 +6.8 box AP——都是*相邻*区间)、扩散增强综述、术语表里的"域差"。
- **为何尚未解决:** 遮挡感知(ODGEN)、小目标遥感(AeroGen)、长尾(X-Paste)都有强增益——但**没有一个隔离出锚点的"小 + 遮挡 + 室内监控"区间**,且对该区间的"有用→有害"比例/过滤拐点未测量。
- **能在锚点上演示?** y —— 用现有 Flux 流水线 + 一个电梯数据集直接可测(扫合成与真实之比,量尺寸相关 AP)。
