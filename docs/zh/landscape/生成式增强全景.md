# 生成式增强全景(generative-augmentation-landscape 中文版)

## 是什么
一张宽扫描地图:生成/合成图像如何被用于增强视觉模型的训练数据,以及从*盲*增强向*诊断驱动、针对性*增强的新兴转变。本节点锚定 Phase 0 全景,向下喂给 `20-loop/` 的主干脊柱。

## 为何重要
它框定了项目的核心赌注:为修复*特定的、诊断出的*模型弱点而生成数据,胜过不加区分地生成。全景分四块:(a) 用什么生成(扩散方法),(b) 为何有效/何时失败(数据为中心的框架、域差),(c) 如何针对(失败归因 + 修复),(d) 必须落到哪里(小目标检测 / 锚点任务)。

## 关键论文
- AI 生成图像作为数据源(Yang 2023)—— 合成数据作为一种来源;每张带标注图像便宜约 47 倍;copy-paste / layout-to-image / 免标注流水线。
- 数据为中心 AI 综述(Zha 2023)—— 数据为中心的框架;以训练数据开发为杠杆。
- 扩散增强综述(Alimisis 2024)—— 扩散增强方法的分类 + 评估指标(stage 3 菜单)。
- 小目标检测综述(Nikouei 2025)—— 小目标检测的挑战(遮挡、类别不均衡)+ 尺寸相关 AP;最贴近锚点。
- SafeFix(Ouyang 2025)—— 诊断→针对性生成→VLM 过滤→重训的闭环;脊柱的已发表模板。

## 开放问题
- 针对性/诊断驱动的增强(SafeFix 式)能否从分类泛化到**检测**?大多数修复工作以分类为中心。
- 如何在检测的粒度(逐目标、逐属性)而非逐图类别上诊断失败?
- 哪种扩散增强方法(语义编辑 vs layout 条件 vs 个性化)最能为小/遮挡目标填上域差?
- 什么样的合成与真实之比、什么样的过滤(基于 VLM?)真正有助而非有害检测 mAP?

## 我们的看法
这个领域已经备齐了零件——便宜的合成数据、一份扩散方法菜单、一个数据为中心的理由、以及一个闭环修复模板(SafeFix)——但闭环大多在分类上被演示。未开采的缝隙是**面向小目标检测的诊断驱动针对性生成**,这恰恰就是锚点任务。GAPS 应聚焦在那条缝隙。
