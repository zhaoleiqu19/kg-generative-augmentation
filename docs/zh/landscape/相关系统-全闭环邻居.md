# 相关系统 — 全闭环与最近邻(related-systems-whole-loop 中文版)

## 是什么
一个先验技术(prior-art)节点,梳理离项目完整"诊断→生成→过滤→重训"闭环最近的*系统*,以便精确说出哪些已经造好、缝隙在哪。loop 脊柱(`20-loop/`)逐阶段地图式综述这个领域,而本节点地图式综述那些组合了多个阶段的**端到端邻居**,并用让检测成为难点的四件事来逐一核对:免费标签、逐目标诊断、切片级指标、廉价重训。

## 为何重要
交付物是 `GAPS.md`。它的可信度取决于:展示最接近的现有系统,并指出每个在哪里达不到"诊断驱动、真实图像、能处理遮挡、用于检测"的增强。本节点就是"这条缝隙确实开着、而不是被我们漏看的某个邻居填上了"的证据。

## 关键论文

### 最接近的全闭环邻居(由某种信号牵引的检测/生成)
- LTDA-Drive(Yurt 2025)—— generate→insert→**LLM 过滤**,用于长尾 3D 检测;KITTI 稀有类 **+34.75%**。缺的那块:由*盲长尾*牵引,而非实测的逐切片诊断。
- GALOT(Hong 2024)—— 明确的 **AL 准则 → 生成**闭环;**分类**、纯合成、文本伪标签(无框)。
- SafeFix(Ouyang 2025)—— 诊断→针对性生成→VLM 过滤→重训;已发表的闭环模板,但以**分类**为中心。
- Chegini(2023)—— 诊断→扩散生成,困难子群 ~21%;**分类**。

### 放置 / 规格邻居("放哪",与 G4 相关)
- Scene-Aware(Petersen 2025)—— 学习式**概率位置模型** + inpainting;**+1.4 mAP**(随机放置基线的 2.8 倍)。放置来自场景先验,而非诊断。
- DIAG(Girella 2024)—— **人标区域**,免训练 in-distribution 缺陷生成;KSDD2 上 **+18–28% AP**。切片→spec 标记的手工版本。

### 树里已有的引擎(由给定 layout 牵引的检测生成)
- ODGEN(Zhu 2024)· GeoDiffusion(Chen 2023)· ReCon(Zhu 2025)· AeroGen(Tang 2024)· X-Paste(Zhao 2023)

### 校准用的怀疑派
- 大数据迷思(Voetman 2023)—— 纯合成检测在简单任务上达到*近持平*(AP 偏差 0.09–0.12)——是替代品,不是放大器。

## 开放问题
- 有没有哪个邻居是从一个**有定位的、逐目标的检测诊断**(而非盲长尾 / 通用 AL 不确定性 / 人工标记)来牵引生成的?——没找到。
- 有没有哪个在**真实图像**上能在**遮挡下产出正确框**?ODGEN/Scene-Aware 最接近(逐目标 / 在场景中 inpaint);GALOT/大数据迷思绕开了框(纯合成 / 人工)。
- 哪个邻居的**过滤器**(LLM agent、VLM、CLIP)能迁移成一个带明确安全比例/停止的检测验证器(→ G3)?

## 我们的看法
闭环已经*碎片化地*存在:带 LLM 过滤的长尾检测生成(LTDA-Drive)、一个真正的 AL→生成闭环(GALOT)、场景感知放置 + inpainting(Scene-Aware)、人标区域的 in-distribution 生成(DIAG)、以及一个分类修复模板(SafeFix)。**没有一个把"有定位的逐目标检测诊断"接到"可控、能处理遮挡、用真实图像的生成器 + 一个明确的验证器"上。** 那个交集就是缝隙——恰是收窄后的 G1(检测全闭环),吃 G2/G4(诊断→spec)和 G3(安全比例/过滤)。
